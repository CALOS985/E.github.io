NotHitBy

临时指定无法击中玩家的hit类型.

必要参数:
value = attr_string 或 value2 = attr_string (字符串型)
上面的参数只有1种能被指定.attr_string应是1个标准的hit属性字符串.见详情.

可选参数:
time = effective_time (整型)
指定NotHitBy属性起作用的游戏帧数.默认是1.

详情:
玩家有2个hit属性槽,能用"value"或"value2"的值来设置给NotHitBy控制器.
这些槽也能被HitBy控制器设置.当设置了1个槽,便获得了一个朝0倒记的计时器.(有效时间)
如果计时器还未计时到0,槽被认为是激活的.玩家能被HitDef击中仅当HitDef中的参数出现在所有当前激活的槽中.
使用NotHitBy控制器设置指定的槽来包含所有hit属性,除了那些被指定在NotHitBy属性字符串中的.

NotHitBy同时也与HitDef的attr参数遥相呼应. 请参阅 HitDef

示例:
;所有攻击都不能击中
[State ]
type = NotHitBy
trigger1 = 1
value = SCA

;不被普通攻击和所有飞行道具击中
[State ]
type = NotHitBy
trigger1 = 1
value = , NA, AP
;attr_string 不写就表示SCA

貌似只有第一组属性槽(SCA)省略,才有效.比如value = ,NA表示对普通攻击免疫,这貌似是准确的.
只要写上SCA就会导致所有攻击都无效.其他还有很多的bug,无法用规律描述出来.比如本人试出来 value = S,NA会导致飞行道具无法击中等等.
